# 安全策略

English version: [SECURITY.md](SECURITY.md)

## 支持版本

本项目仍处于 pre-1.0 阶段。除非后续创建 release branch，安全修复默认应用到 `main` 最新提交。

## 报告漏洞

请不要在维护者有时间审查和响应前公开漏洞细节。

在可行时，请私下向维护者报告问题。在正式安全联系渠道配置前，请通过私有维护者讨论或直接联系仓库所有者。

相关问题包括：

- secrets 泄露或不安全默认配置
- 私有钱包数据暴露
- 服务端请求伪造
- 不安全的文件导入/导出行为
- 具有实际利用路径的依赖漏洞
- 可能误导用户理解 evidence 或 confidence 的行为

## 项目安全边界

Wallet Map 不应：

- 要求私钥或助记词
- 签名交易
- 自动化钱包操作
- 在仓库中存储真实 API key
- 默认上传钱包地址集合
