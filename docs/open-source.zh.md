# 开源规范

English version: [open-source.md](open-source.md)

## 项目边界

Wallet Map 适用于：

- 个人链上足迹审计。
- 钱包关系研究。
- 合规友好的调查和复盘。
- 围绕链上图分析的开源教育。

Wallet Map 不适用于：

- 私钥处理。
- 交易签名。
- 自动化钱包操作。
- 规避平台规则。
- 批量滥用或垃圾操作流程。

## 隐私

默认本地优先。

- 默认不上传地址集合。
- 不记录 API key。
- Fixture 不存储私有 secrets。
- 导出由用户控制。
- 分享报告前应提供或使用脱敏能力。

## 文档语气

文档应实用、正式、克制。

- 说明工具能推断什么、不能推断什么。
- 保持示例可复现。
- 清晰标注合成数据。
- 避免夸大确定性的说法。

## 贡献预期

贡献应：

- 遵守 package 边界。
- 为新行为添加测试。
- 避免无关重构。
- 保持 fixture 小而可检查。
- 行为变化时更新文档。

## 安全报告

遵循仓库 [安全策略](../SECURITY.zh.md)。在正式安全联系渠道建立前，请优先私下联系维护者，不要提前公开漏洞、泄露 secrets 或不安全默认配置细节。
